# ESP32-C3 + ExpressLRS BLE Joystick

Приём каналов пульта **Radiomaster Pocket** (EdgeTX) с модулем **ExpressLRS 2.4** по **BLE** на **ESP32-C3**. Вывод первых 12 каналов в формате **0–255** в Serial.

## Требования

- Плата: **ESP32-C3**
- Среда: **Arduino IDE** или **PlatformIO** с поддержкой ESP32
- Библиотека: **ESP32 BLE Arduino** (обычно идёт с ядром esp32)

## Подключение

1. На пульте включите режим **BLE Joystick** (ExpressLRS).
2. Прошейте скетч на ESP32-C3, откройте **Serial Monitor** (115200 бод).
3. ESP32 найдёт устройство «ExpressLRS Joystick», подключится и начнёт выводить строки вида:
   ```
   CH1:127 CH2:127 CH3:0 CH4:128 CH5:0 CH6:0 CH7:0 CH8:0 CH9:0 CH10:0 CH11:0 CH12:0
   ```

## Важно про переподключение

После перезагрузки ESP32 или обрыва связи пульт может не возобновлять отправку пакетов. В этом случае **один раз перезапустите BLE Joystick на пульте** (выкл/вкл в настройках ExpressLRS). В Serial при необходимости появится подсказка.

## Структура проекта

| Файл | Описание |
|------|----------|
| `esp32_c3_joystick.ino` | Основной скетч: BLE, вывод каналов в Serial |
| `ble_connection.h` | Подключение к пульту по BLE, подписка на HID Report |
| `joystick_handler.h` | Разбор пакетов ExpressLRS BLE, каналы 1–12 (0–255) |

## Отладка

- **BLE:** в `ble_connection.h` задайте `#define BLE_DEBUG 1` для подробного лога с метками времени.
- **Джойстик:** в `joystick_handler.h` задайте `#define JOYSTICK_DEBUG 1` для вывода сырых пакетов (HEX) в первые приёмы.

## Формат каналов

- **CH1–CH8** — аналоговые оси (0–255).
- **CH9–CH12** — кнопки (0 или 255).

В основном скетче массив каналов получается через `getChannelValues(uint8_t ch[12])`; проверка наличия данных — `hasJoystickData()`.
